# 需求词标准驱动的需求挖掘与雷达优化设计

## 目标

以《重点游戏工具&攻略玩家需求词监控.xlsx》为唯一业务词表标准，重构工具需求挖掘、需求雷达和相关采集筛选逻辑，优先解决当前结果泛化、跨游戏污染、重点词遗漏和模型判断不稳定的问题。

第一版纳入：

- 7 款重点游戏、240 个游戏专属需求词。
- 39 个全品类通用需求词。
- 一级核心词、二级长尾词、三级热点词三类优先级。
- 原词与代码内固定近义词扩展。

## 已确认规则

### 重点游戏

重点游戏包括：

- 三角洲行动
- 失控进化
- 异环
- 洛克王国世界
- 原神
- 鸣潮
- 崩坏：星穹铁道

重点游戏仅匹配：

1. 当前游戏的专属词；
2. 全品类通用需求词。

不得使用其他游戏的专属词，禁止跨游戏匹配和串线。

### 非重点游戏

非重点游戏仅匹配 39 个通用需求词，不再使用大模型自由发现新的需求方向。

### 一级词

- 定义：高频高流量核心需求词。
- 单条独立内容首次命中后，直接生成“重要”雷达线索。
- 不自动进入正式需求排行。
- 后续由人工确认或升级为正式需求。
- 多来源或多内容验证用于增加证据、分数和排序，不重复生成线索。

### 二级词

- 定义：稳定流量长尾需求词。
- 第一条独立内容命中时生成“观察”雷达线索。
- 同游戏、同标准需求词出现第二条独立内容后升级为“重要”。
- 不自动进入正式需求排行。
- 独立内容以不同平台内容 ID 或不同来源 URL 判断，重复采集快照不计为第二条。

### 三级热点词

- 定义：随版本或活动变化的临时热点需求词。
- 仅对发布时间在最近 7 天内的内容生效。
- 单条命中即生成“重要”雷达线索。
- 同时命中明确版本号、活动时间、上线/下线时间、加强、削弱、BUG、修复等节点时升级为“紧急”。
- 超过 7 天的历史内容可保留入库，但不生成新的热点提醒。

## 固定近义词方案

系统采用“原词 + 固定近义词扩展”，不在运行时调用大模型生成近义词。

近义词由代码词库维护，分为两层：

### 通用表达映射

用于覆盖常见搜索表达差异，例如：

- 查询：查询、查看、查、在哪看
- 计算器：计算器、计算、测算、估算
- 模拟器：模拟器、模拟、预演
- 推荐：推荐、选择、怎么选、哪个好
- 搭配：搭配、配队、组合、配置
- 排行：排行、排名、榜单、强度榜
- 点位：点位、位置、地点、在哪
- 路线：路线、路径、跑图路线、规划
- 材料：材料、素材、养成资源
- 攻略：攻略、教程、打法、怎么过

### 标准词专属别名

对游戏术语、缩写和玩家惯用说法维护专属别名，例如：

- 3X3任务：3X3、三乘三、3×3任务
- 卡战备：卡战备、压战备、战备限制
- 每日密码：每日密码、今日密码、密码门
- 圣遗物评分器：圣遗物评分、圣遗物打分
- 声骸评分器：声骸评分、声骸打分

所有别名必须显式写入代码配置并接受测试，不允许自动语义扩展。匹配结果始终归一到 Excel 中的标准需求词。

## 统一词库模型

新增一个代码内统一词库模块，负责提供：

- 标准游戏名及固定别名；
- 标准需求词；
- 需求类别；
- 优先级；
- 固定近义词；
- 是否为通用词；
- 建议工具类型；
- 热点有效期。

建议结构：

```python
DemandKeywordRule(
    game_name="三角洲行动",
    category="工具箱工具核心词",
    priority="level_1",
    canonical_term="战备搭配",
    aliases=("战备搭配", "配战备", "战备怎么配"),
    suggested_tool_type="配装/战备工具",
)
```

Excel 内容转换为版本化代码数据，运行环境不依赖用户下载目录中的 Excel 文件。Excel 仍作为业务源文件，代码词库作为发布时的可执行快照。

## 匹配流程

每条已关联游戏的内容执行以下流程：

1. 根据游戏名称和固定别名确定当前游戏。
2. 读取当前游戏专属词与通用词；非重点游戏只读取通用词。
3. 对标题、正文、命中词和可用高赞评论进行统一规范化。
4. 使用最长词优先策略匹配原词和固定近义词。
5. 多个别名指向同一标准词时，只产生一次标准命中。
6. 同一内容可命中多个不同标准需求词。
7. 根据标准词优先级和独立证据数量生成或更新雷达线索。
8. 将标准词、命中别名、原始片段、来源内容和分级理由写入证据。

规范化仅处理：

- 大小写；
- 全角/半角；
- 常见空格和标点；
- `3×3`、`3X3` 等等价字符。

不得采用模糊编辑距离匹配，避免短词和相似游戏术语误命中。

## 线索合并与跨游戏隔离

线索稳定签名调整为：

```text
游戏ID + 标准需求词
```

同游戏、同标准需求词的不同别名合并为一条线索，并累计独立证据。

不同游戏即使标准词相同，例如“互动地图”，也必须分别生成线索。

通用词命中后仍归属于内容当前关联的游戏，不能生成“未知游戏”或跨游戏公共线索。

## 评分与排序

规则词优先级成为主要排序依据：

- 一级词基础分：70，默认重要。
- 二级词基础分：40，默认观察；第二条独立证据提升至 60，升级重要。
- 三级热点词基础分：65，默认重要；明确时效节点提升至 85，升级紧急。

在基础分上叠加：

- 第二个独立来源：+8
- 第三个及以上独立来源：最多再加 +7
- 互动突增：最高 +10
- 当前游戏为重点游戏：+5

最终分数封顶 100。互动和来源只影响排序与升级，不改变标准需求词。

## 大模型职责调整

大模型不再负责：

- 自由判断需求方向；
- 发明新需求词；
- 生成近义词；
- 决定一级、二级、三级优先级；
- 将未命中标准词的内容写入需求雷达。

大模型仅负责：

- 总结已命中内容表达的具体问题；
- 合并同一标准词下多个证据的用户诉求；
- 提炼正式需求标题和说明；
- 解释为什么适合某类工具或攻略；
- 在人工升级正式需求时补充结构化分析。

模型失败不影响规则命中线索的生成，只将“模型总结”标记为待补充。

## 采集策略

采集关键词由统一词库生成：

- 重点游戏：游戏名与一级词组合优先采集；二级词按较低频率采集。
- 通用词：与游戏名组合采集，禁止只用通用短词后将结果复制关联到所有游戏。
- 三级热点词：与游戏名组合进行近 7 天高频采集。
- 原有纯游戏名探索通道保留用于内容覆盖，但未命中标准词的内容不进入需求雷达。

平台不支持组合搜索时，采集结果必须在入库前确认内容属于当前游戏。

## 正式需求

词表命中只生成雷达线索，不自动创建正式需求。

人工点击“升级需求”后：

- 正式需求标题以标准需求词为核心；
- 继承全部独立证据；
- 继承优先级、评分和建议工具类型；
- 由大模型生成摘要，但不得改变标准需求方向；
- 同游戏、同标准词升级操作保持幂等。

## 首页与雷达展示

线索卡片明确展示：

- 游戏；
- 标准需求词；
- 命中的原始表达；
- 一级词、二级词或三级热点词；
- 工具类、攻略类或热点类；
- 当前独立证据数；
- 升级原因；
- 原始内容链接。

支持按游戏、优先级、需求类别和标准词筛选。

## 历史数据处理

上线时对现有内容执行一次无提醒回扫：

- 建立标准词命中和证据基线；
- 不将全部历史一级、二级词集中推送为新线索；
- 最近 7 天三级热点词可按当前规则生成提醒；
- 历史已有雷达线索若无法归一到标准词，则从待处理雷达中归档，不删除原始内容。

## 数据维护

第一版词库固定在代码中，不提供后台编辑。

后续业务调整流程：

1. 更新业务 Excel；
2. 同步更新代码词库；
3. 运行完整性、重复词和跨游戏隔离测试；
4. 提交并发布。

系统启动时不直接读取本地 Excel，避免部署环境依赖和人工文件被意外修改。

## 测试与验收

必须覆盖：

- 7 款重点游戏的 240 个专属词全部可加载。
- 39 个通用词全部可加载。
- 重点游戏不能命中其他游戏专属词。
- 非重点游戏只能命中通用词。
- 原词和固定别名归一到同一标准词。
- 一级词首次命中生成重要线索。
- 二级词首次为观察、第二条独立内容升级重要。
- 重复采集同一内容不计为第二条证据。
- 三级热点词仅对近 7 天内容生效。
- 三级词与明确版本或时效节点共同出现时生成紧急线索。
- 一条内容可生成多个不同标准词线索。
- 未命中标准词的模型输出不能创建雷达线索。
- 同标准词在不同游戏中保持隔离。
- 升级正式需求幂等且证据完整。
- 现有正式需求排行、需求管理和外部同步不被破坏。

最终验收标准：

- 规则命中结果可以追溯到 Excel 标准词和具体命中别名。
- 重点游戏不存在跨游戏需求词污染。
- 非重点游戏不再出现模型自由发明的需求方向。
- 大模型不可绕过词库直接创建雷达线索。
- 业务方可以通过修改代码词库稳定复现相同判断结果。
